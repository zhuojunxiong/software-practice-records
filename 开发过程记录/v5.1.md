## 二、AI 智能搜索

### 2.1 用户需求

> **用户**：我们现在开始做 v5.1，能不能加上 AI 智能搜索，就是即使我的搜索词写的不好，你也可以理解我的意思并且帮我找到我需要的文章。我可能只是在搜索词描述我的需求，你也可以把我需要的文章找出来。

> **AI 助手**：好的，这是一个很好的功能需求。让我先全面了解项目的架构和现有搜索功能，再给你制定方案。

AI 助手先深入探索了项目结构，确认了：
- 现有搜索是 SQL ILIKE 关键字匹配（`app/article/services.py:36-55`）
- 已有 AIService 和 OpenAI 兼容 API 调用能力（`app/ai/services.py`）
- 搜索模板已标注"AI 搜索"但实际不是 AI 驱动
- 前端已有 loading 状态基础设施

### 2.2 方案设计

> **AI 助手**：我设计了**三阶段 AI 搜索管道**方案：

```
用户输入 → [阶段1] AI 意图理解 + 关键词扩展
       → [阶段2] SQL 多关键词搜索（最多 20 篇候选）
       → [阶段3] AI 语义重排序 + 生成推荐理由
       → 返回结果（含 AI 理解卡片 + 每篇推荐理由 + 相关性评分）

任何阶段失败 → 自动降级到传统 ILIKE 搜索
```

> **用户**：批准了方案。

> **AI 助手**：开始实现。先改 AI 服务层，新增两个场景配置 `search_intent` 和 `search_rerank`，然后新增 6 个方法：
> - `search_intent_and_expand(query)` — 阶段1
> - `_sql_multi_keyword_search(keywords)` — 阶段2
> - `rerank_with_reasons(query, understanding, candidates)` — 阶段3
> - `smart_search(query, page, per_page)` — 主编排
> - `_parse_json_dict()` / `_parse_json_list_of_dicts()` — 容错解析

AI 助手依次修改了 6 个文件，完成了完整的前后端实现。

### 2.3 搜索失败的排查

> **用户**：这个搜索功能还是有问题，输入之后什么也没得出来。

> **AI 助手**：让我查一下。发现问题有两个：
> 1. `fallback: True` — AI 阶段3 失败了。原因是 Flask 进程没走代理，连不上 DeepSeek API
> 2. 首页搜索成功后跳转到搜索页的逻辑有 bug —— 首页没有结果展示容器

> **AI 助手**：修复了两个问题：在 `.env` 里加了 `HTTP_PROXY` 和 `HTTPS_PROXY` 配置，前端加了首页自动跳转到 `/search?q=xxx&ai=1` 的逻辑。重启后验证 `fallback: False`，三阶段管道全部正常。

## 四、启动体验优化

### 4.1 .env 自动加载

> **用户**：所以以后日常启动只需要一条命令 API 也可以正常了？

> **AI 助手**：对，就是一条命令 `.venv\Scripts\python run.py`。`.env` 里的 API Key 和代理配置会自动加载。

AI 助手做了以下改动：
- `run.py` 新增 `python-dotenv` 自动加载 `.env` 文件
- 创建 `.env.example` 配置模板
- 放宽 `requirements.txt` 版本以兼容国内镜像
- 新增 `python-dotenv` 到依赖

### 4.2 一键启动脚本

> **AI 助手**：日常启动就一条 `.venv\Scripts\python run.py`，或者直接双击 `启动项目.bat`。API Key 和代理都在 `.env` 里配好了自动生效。

> **用户**：怎么直接弹出来一个记事本了？

> **AI 助手**：Windows 把 `.bat` 当成文本文件了。在终端里跑就行。

最终 `启动项目.bat` 只保留 4 行核心逻辑：切换目录 → 启动 Python → 暂停。

### 4.3 启动项目

> **用户**：我电脑里面有合适的虚拟环境吧，帮我启动项目。

AI 助手尝试在终端安装依赖时遇到了网络代理问题——pip 被 Windows 系统代理拦截。经过排查发现用户电脑配置了 `127.0.0.1:7890` 代理（Clash/V2Ray 类工具），但当前 bash 会话没有正确设置环境变量。最终通过 `export http_proxy="http://127.0.0.1:7890"` 解决了依赖安装问题。

依赖安装完成后，AI 助手初始化了数据库、加载了演示数据、启动了 Flask 服务器。

### 4.4 AI 接口失败

> **用户**：怎么写着 AI 接口失败了？

> **AI 助手**：问题找到了——Flask 服务没有走代理，连不上 DeepSeek。已经在 `.env` 里加了 `HTTP_PROXY` 和 `HTTPS_PROXY`，重启后 `fallback: False`，三阶段管道全通。

## 六、关键决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| AI 搜索架构 | 三阶段管道 | 平衡准确性和性能，每阶段独立可降级 |
| 搜索端点权限 | 公开（无需登录） | 搜索是公共功能，不应限制 |
| 搜索结果展示 | JSON API + JS 动态渲染 | 保留服务端渲染降级路径 |
| 中文引号处理 | 统一用「」替代 | 避免 Python 字符串解析冲突 |
| 版本号策略 | V5.1（非 V0.5.1） | 用户指定 |
| requirements.txt 版本 | `>=` 最低版本 | 兼容国内镜像源 |
| 代理配置 | 写入 .env | 自动加载，无需每次 export |
