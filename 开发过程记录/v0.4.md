## 一、当前问题：页面能运行，但不像现代产品

**用户：**  
现在的前端界面很不好看。页面里的导航、按钮、搜索框、登录注册入口都像浏览器默认样式，只是横线加汉字，看起来像很早期的网页。排版也不均衡，没有美感。现在应该怎么让开发助手改良 UI？我需要详细描述自己心目中的前端设计吗？

**AI：**  
当前问题不是“颜色不够好看”，而是还没有真正进入 UI 设计阶段。  
现有页面主要问题包括：

1. 顶部导航、登录注册、搜索按钮基本还是浏览器默认样式。  
2. 页面缺少统一的视觉系统，按钮、输入框、卡片、标签、表单没有组件化。  
3. 首页虽然有口号，但没有形成“多人知识创作社区”的产品感。  
4. 页面结构不完整，缺少现代产品常见的 Hero 区、文章卡片、专栏卡片、分类入口、页脚等。  
5. 现在不应该随便加功能，而应该先让已有功能看起来像一个完整产品。

AI 建议：需要描述前端设计方向，但不需要一开始写成完整设计稿。更重要的是给开发助手明确：  
V0.4 的目标是前端 UI/UX 产品化改造，不是新增 AI 功能。

**用户：**  
我们现在就是在做 V0.4，AI 功能到 1.0 再设计。你告诉我现在应该怎么和开发助手讲。

本版本只做前端 UI/UX 产品化改造，不重写项目，不破坏后端功能，不接入任何 AI API。

## 三、为什么别人用开发助手做出来的前端更好看

**用户：**  
为什么别人用开发助手做出来的前端那么好看？是不是我哪里做错了？应该怎么反思？

**AI：**  
别人做出来的前端好看，不是因为开发助手天生更懂审美，而是因为他们给开发助手的输入更像“设计交付物”，而不是一句“帮我美化”。

之前的问题在于：  
虽然已经说了“现代、清爽、有知识社区感”，但这仍然偏抽象。开发助手很难凭几个形容词做出稳定好看的 UI。

正确做法应该是：

1. 给当前页面截图，让开发助手先诊断哪里不好。  
2. 给参考图，让它知道目标风格。  
3. 让它先提炼设计规则，而不是直接写代码。  
4. 建立 `DESIGN_SPEC.md`，形成长期前端规范。  
5. 按页面分阶段改造，而不是一次性全改。  
6. 运行项目，用浏览器截图验收。

AI 反思：  
之前给出的提示词更像功能需求说明，还不够像真正的 UI 任务。应该把任务改成：

```text
先诊断当前 UI 问题，再建立设计系统，再按页面重构，再用浏览器截图验证，再总结验收结果。
```

## 四、除了提示词，还可以做什么

**用户：**  
除了给提示词之外，我还可以做些什么？

**AI：**  
除了提示词，还应该做六件事：

### 1. 给开发助手看当前页面截图

不要只说“丑”，要明确截图里的问题，例如：

- 顶部导航是默认链接样式；
- 登录、注册、搜索按钮像原生控件；
- 页面上方和 Hero 区割裂；
- 字体层级生硬；
- 搜索框、按钮、导航、链接没有组件化。

### 2. 准备 3 到 5 张参考图

参考图可以包含：

- 简洁知识管理类页面：学习留白和克制感；
- 内容阅读类页面：学习文章页排版；
- 专栏社区类页面：学习多人内容结构；
- 用户主页类页面：学习平等用户体系；
- 好看的博客首页：学习文章卡片和首页布局。

重点不是照抄，而是学习：

- 留白；
- 卡片；
- 字体层级；
- 导航栏；
- 首页结构；
- 阅读体验。

### 3. 创建 `DESIGN_SPEC.md`

让开发助手在项目根目录创建长期设计规范文件，写清楚：

- 产品定位；
- 视觉关键词；
- 禁止风格；
- 页面最大宽度；
- 颜色变量；
- 字体层级；
- 间距规范；
- 按钮规范；
- 输入框规范；
- 卡片规范；
- 标签规范；
- 导航栏规范；
- 响应式规范；
- 每个页面的 UI 验收标准。

### 4. 准备真实内容

页面不好看有时不是 CSS 问题，而是内容太假。  
可以准备更真实的文章标题、专栏名称、分类和标签，例如：

- 如何从第一性原理重新理解博客系统；
- 为什么写作是知识整理的最好方式；
- 一个多人知识专栏平台应该如何设计；
- 软件工程实践记录；
- 产品设计与需求分析；
- 知识管理方法；
- 软件工程、需求分析、知识管理、产品设计、Web 开发等标签。

### 5. 分阶段改造

不要让开发助手一次性改完整个项目。建议分为：

1. 全局设计系统、公共模板、导航栏、按钮、输入框、卡片基础样式；  
2. 首页；  
3. 文章列表页和专栏页；  
4. 文章详情页和评论区；  
5. 登录、注册、写文章页；  
6. 响应式、hover、focus、空状态和细节修复。

### 6. 用截图验收

不能只看开发助手的文字总结。  
每次改完都应该运行项目，看真实浏览器效果，并截图继续反馈。

## 五、是否可以给开发助手网页，让它自己学习

**用户：**  
我用的是编辑器里的开发助手，能不能给它网页，让它自己去学？

**AI：**  
可以给网页链接，但不能指望它“自己上网学习审美，然后自动变成设计师”。  
更靠谱的做法是：

1. 给参考网页链接；
2. 给参考网页截图；
3. 给当前项目截图；
4. 要求开发助手先提炼设计规则；
5. 再按这些规则改造项目。

建议给开发助手的说法：

```text
下面这些网页和截图是 UI 风格参考，不是让你照抄。

请你先观察参考页面的设计特点，并提炼成适合本项目的 DESIGN_SPEC.md。
你需要学习的是页面结构、导航方式、字体层级、留白比例、卡片样式、按钮样式、文章列表组织方式、专栏页组织方式和响应式处理方式。

请不要直接复制任何参考网站的品牌元素、文案、Logo 或专属视觉。
```

结论：  
不是“给一个网址，让它自己学”，而是“给截图和参考链接，让它反向提炼设计规范，再按规范重构”。
