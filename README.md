# 软件综合实践项目记录

本仓库用于整理“个人博客系统”实训项目的前期分析、课程要求、技术选型、项目计划和 AI 协作记录。内容已按公开学习材料的标准进行整理：尽量保留分析过程，弱化个人信息，统一使用“AI”作为相关工具的泛称。

## 项目逻辑构成

```text
课程要求理解
  -> 实训总体任务分析和初步开发计划/

需求与产品定位
  -> 需求分析/

教材示例与技术路线分析
  -> 示例代码和书本教程分析/

每日过程记录
  -> 00-第一天工作记录.md
```

## 文件命名规则

- 使用“编号 + 主题”的形式，避免“ai聊天”“ai对话2”等临时文件名。
- Markdown 文件统一使用 `.md` 后缀。
- 涉及 AI 协作的材料统一使用“AI”表述，不保留具体产品名称。
- 涉及个人身份、课程交流渠道、平台入口等信息时，只保留项目所需的抽象描述。
